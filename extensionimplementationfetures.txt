Cut & Click — Extension & Advanced Features Plan (Phase 11-20)
Detta dokument beskriver nästa steg för Cut & Click för att gå från en stabil MVP till en premium, polerad produkt med fokus på användarupplevelse, "emotional design" och djupare integrationer.

Mål med utökningen:
- Erbjuda en sömlös profilhantering med riktig lagring av bilder.
- Öka kundlojaliteten genom ett fungerande favoritsystem.
- Modernisera inloggningen med sociala leverantörer (Google/Apple).
- Förbättra datakvaliteten med adressvalvidering och tvingande mediauppladdning.
- Skapa en "magisk" känsla genom subtila animationer och mikrointeraktioner.

Fas 11 — Profile Picture & Storage Integration (Real Media)
──────────────────────────────────────────────────────────────────────────────
Mål:
Gå ifrån text-baserade bild-URL:er till riktig filuppladdning via Supabase Storage. Detta gäller både kundens profilbild (avatar) och salongens profil/omslagsbild.

11.1 Supabase Storage Setup (Infrastruktur)
- Skapa en publik bucket i Supabase Dashboard: `avatars`.
- Konfigurera CORS i Supabase för att tillåta din domän (localhost i för tillfället).
- Sätt RLS-policies för bucketen (SQL):
```sql
-- Tillåt alla att SE profilbilder
CREATE POLICY "Public Access" ON storage.objects FOR SELECT USING (bucket_id = 'avatars');

-- Tillåt inloggade användare att ladda upp till sin egen mapp
CREATE POLICY "User Upload" ON storage.objects FOR INSERT 
WITH CHECK (bucket_id = 'avatars' AND (storage.foldername(name))[1] = auth.uid()::text);

-- Tillåt uppdatering/borttagning av egna bilder
CREATE POLICY "User Update" ON storage.objects FOR UPDATE 
USING (bucket_id = 'avatars' AND (storage.foldername(name))[1] = auth.uid()::text);
```

11.2 Frontend — Smart ImageUpload-komponent
Skapa en premium-komponent för bilduppladdning som hanterar beskärning och kompression.
- Arkitektur: `client/src/components/common/ImageUpload.tsx`
- Funktioner:
  * Drag & Drop yta med SVG-ikoner (Lucide-React: UploadCloud).
  * Förhandsgranskning före uppladdning med `URL.createObjectURL`.
  * Automatisk rensning av gamla filer från storage vid ny uppladdning.
  * Hantering av maximal filstorlek (2MB) med varnings-toast.
- Props: `onUploadSuccess`, `aspectRatio`, `folderPath`.

11.3 Hover Effect & "Change Photo" Interaction
När en användare hovrar över sin profilbild (den runda cirkeln i inställningarna):
- Bilden ska förmörkas något med en mjuk overlay (`bg-black/40`).
- En "Byt bild"-ikon (Camera ikon) visas centrerat med en mjuk fade-in.
- Framer Motion: Använd `whileHover={{ scale: 1.05 }}` och en transition på 0.2s.
- CSS: `opacity: 0` → `opacity: 1` på overlayen vid hover (`group-hover`).

11.4 Backend-synkronisering
- Vid lyckad uppladdning: fetch-anrop till `PUT /api/auth/profile` för att spara den nya sökvägen.
- Backend-kontroll: Verifiera att filen faktiskt finns i storage innan databasen uppdateras.
- Controller-logik: `const { error } = await supabase.from('users').update({ profile_pic_url: path }).eq('id', userId)`.

11.5 Error Handling & Rollbacks
- Om DB-uppdateringen misslyckas efter filuppladdningen: Ta bort filen från Storage för att undvika "ghost files".
- Visa en toast: "Kunde inte spara profilbilden, försök igen."

Klart när:
[ ] Supabase Bucket `avatars` är konfigurerad med RLS.
[ ] Man kan ladda upp bild via drag-and-drop i inställningar.
[ ] Hover-effekt med "Byt bild" fungerar sömlöst på runda profiler.
[ ] Bilden sparas permanent och visas vid inloggning.

Fas 13 — Social Auth & User Onboarding (Future Proofing)
──────────────────────────────────────────────────────────────────────────────
Mål:
Förbereda plattformen för Google och Apple inloggning via Supabase för att sänka tröskeln för nya användare.

13.1 Supabase Provider Setup (External Integration)
- Steg-för-steg guide för aktivering:
  1. Hämta Google Client ID & Secret från Google Cloud Console.
  2. Hämta Apple Team ID, Key ID och ladda upp `.p8` filen till Supabase Auth Dashboard.
  3. Konfigurera Redirect URL till `https://cutandclick.se/auth/callback`.
  4. Aktivera "Sync with database" så att varje ny social login skapar en rad i `users` tabellen automatiskt via en Supabase Trigger.

13.2 Frontend — Social Button Group Component
- Skapa en komponent `SocialAuthGroup.tsx` som används i både Login och Register.
- Branding:
  * Google: Vit knapp, Google G-logga, "Fortsätt med Google".
  * Apple: Svart knapp, vit Apple-logga, "Fortsätt med Apple".
- Logic: `supabase.auth.signInWithOAuth({ provider: 'google' })`.

13.3 "Missing Data" Flow & Forced Onboarding
Eftersom social auth ofta bara ger e-post och namn, måste en ny flödeslogik skapas:
- Om `user.role` är NULL (ny social användare):
  * Tvingande redirect till `/onboarding`.
  * Middleware i React som förhindrar navigering till Home innan roll och stad är satta.
- Onboarding-steg:
  1. Välj roll (Kund / Frisör).
  2. Sätt Stad (Location).
  3. Acceptera användarvillkor.

13.4 Database Triggers (The Glue)
```sql
-- Automatisk profilskapande vid OAuth-login
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.users (id, email, username, role)
  VALUES (new.id, new.email, new.raw_user_meta_data->>'full_name', NULL);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

Klart när:
[ ] Knappar för Google/Apple finns i UI och leder till OAuth-flödet.
[ ] Auth-redirect flödet (`/auth/callback`) fungerar logiskt.
[ ] Onboarding-flödet fångar upp användare med ofullständiga profiler.

Utökad API Dokumentation (Phases 11-20)
──────────────────────────────────────────────────────────────────────────────
Här beskrivs de nya endpoints som krävs för fas 11-20.

GET /api/favourites
- Hämtar inloggad användares alla favoriter.
- Returnerar: `Array<{ barber_id, salon_name, address, profile_image }>`.

POST /api/favourites/:barberId
- Lägger till/tar bort favorit (Toggle).
- Returnerar: Status 201 (Created) eller 204 (Deleted).

PUT /api/barber/settings
- Uppdaterar salongs-info.
- Payload: `JSON { name, bio, phone, address, gallery_images }`.

POST /api/media/upload
- Endpoint för att ladda upp temporära bilder för validering innan de sparas permanent.
- Middleware: `multer` eller direkt integration med Supabase Storage.

GET /api/barber/analytics
- Hämtar statistik för en salong (Antal besök, bokningsfrekvens).

PUT /api/user/onboarding
- Sätter roll och detaljer för OAuth-användare.

──────────────────────────────────────────────────────────────────────────────
Säkerhet & Validering (Hardening)
──────────────────────────────────────────────────────────────────────────────
1. RLS (Row Level Security) - Djupdykning
- Ingen användare ska kunna se en annan användares favoriter.
- Barberare kan endast redigera sin egen `barber_profiles` rad.
- Implementera `CHECK (auth.uid() = user_id)` på alla policies.

2. Rate Limiting
- Implementera `express-rate-limit` på `/api/auth` och `/api/media/upload`.
- Max 5 bilduppladdningar per minut per IP-adress för att förhindra spam.

3. Image Compression (Backend Side)
- Använd `sharp` i Node.js för att komprimera alla inkommande bilder till 1200px bredd max.
- Behåll 85% kvalitet för att balansera hastighet och estetik.
- Konvertera automatiskt till `.webp` för bästa prestanda.

──────────────────────────────────────────────────────────────────────────────
Mobil-strategi & Offline Support (The App Feel)
──────────────────────────────────────────────────────────────────────────────
För att Cut & Click ska kännas som en native-app även när den körs i webbläsaren.

1. PWA Integration (Progressive Web App)
- Konfigurera `vite-plugin-pwa` för att tillåta installation på hemskärmen.
- Skapa en `manifest.json` med temafärger (Pitch Black för statusfältet).
- Implementera en Service Worker som cachar de viktigaste sidorna (Home, Bookings).

2. Offline-logik
- Om nätverket försvinner ska användaren fortfarande kunna se sina kommande bokningar.
- Implementera `react-query` med `persistQueryClient` för att spara data i `localStorage` eller `IndexedDB`.
- Visa en subtil "Offline-indikator" i headern istället för att låta appen krascha.

3. Touch Optimeringar
- Implementera "Pull-to-refresh" på bokningssidan.
- Använd `framer-motion` för att göra listor "scroll-bounce"-vänliga som på iOS.
- Minimera klick-latens genom att använda `active:scale-95` på alla touchable ytor.
